How It Works

Blockchain

The only technology which guarantees the storage of your transactional data in an immutable manner. All data is recorded as transactions and grouped in blocks. These go one by one in a linear manner, forming a chain of blocks. Because of links to previous blocks and cryptography algorithms previously added information cannot be changed. The usage of Merkle trees provides auditability and accountability of blockchains. Thus, a user can verify inclusion of his transaction into a blockchain by asking for proof, which can be done without disclosing information on other transactions. Exonum can operate as either a private or permissioned public blockchain.

BFT consensus algorithm

Byzantine fault tolerant algorithms acknowledge that some actors may behave incorrectly (because of either intentional or unintentional changes). They guarantee the system will operate correctly during maintenance periods or even if under attack. BFT consensus algorithms provide high throughput and transaction settlement usually in under several seconds.

Benchmarks demonstrate up to 3500 tps per one core and 2.5 sec transaction settlement time.

Bitcoin Blockchain anchoring

Anchoring is the process of periodically saving to a public source a fingerprint of system state. This approach negates the need for users to trust the system operator. Exonum is unique in using its' approach which enhances the security of already cryptographically proven systems and creates perfectly trustless environment.
